# Conflux 产品文档

文档基线：`v1.0 稳定化阶段 / Roadmap Solidified`

## 1. 产品定位

Conflux 是一个面向高频思考者的 Agentic Knowledge Flow 工具。

它不以文件夹树组织知识，而是把每一条内容都视作可被重组、可被引用、可被回流更新的扁平知识块。当前产品的核心命题不是“做另一款笔记软件”，而是验证以下三件事是否能同时成立：

- 扁平知识块可以通过多维标签、引用关系与运行时推导形成可持续组织。
- AI 应只承担克制、局部、可确认的任务，而不是吞掉整篇上下文做重型分析。
- `/feed`、`/write`、`/graph` 三个视角可以构成一个低压迫、可扩展、可演进的知识流闭环。

## 2. 版本校正结论

- `v0.1 ~ v0.7` 可以视为相对线性的早期里程碑。
- `v0.8`、`v0.9`、`v1.0` 不是历史上严格切开的独立发布包，而是对已经合入代码主链能力的回顾性命名。
- 当前代码基线已经同时包含 `v0.9` 的段落级本地推荐漏斗，以及 `v1.0` 的长文语义切块与线程串联。
- 因此当前项目的真实称呼仍然是：`v1.0 稳定化阶段`。
- `package.json` 中的 `1.0.0` 现在表示开源发布包版本元数据；更细粒度的产品里程碑与阶段真相仍以 [`04-CHANGELOG.md`](./04-CHANGELOG.md) 为准。

## 3. 当前产品基线

截至当前代码状态，Conflux 的现实基线如下：

- 视觉基线是 `Zen Canvas`：浅灰背景、白色卡片、大留白、低压迫交互。
- 数据基线是扁平 `fluxBlocks`，不引入文件夹树，不引入 `parentId`。
- AI 基线遵循“本地预筛 -> 隐蔽提示 -> 用户确认 -> 精确调用模型”的漏斗。
- 超长输入默认不允许直接落成单一巨型 block，而要先经过本地切块与线程串联。
- `stage / source` 已进入五维模型，并已在 `/write`、`/feed`、`/graph` 与侧边栏完成第一轮低存在感接入。
- 当前已经具备 onboarding seed、本地任务记录、版本历史、回滚与来源追溯；其中 seed 仅在首次冷启动且本地尚无持久化工作区时注入，后续不再干预用户数据。

## 4. 核心对象

### 4.1 Conflux Block

`block` 是 Conflux 的最小知识单元。它是独立对象，不承担树状父子关系。

当前核心字段包括：

- `id`
- `title`
- `content`
- `dimensions`
- `createdAt`
- `updatedAt`
- `revisions`

其中 `dimensions` 当前规范为：

- `domain`
- `format`
- `project`
- `stage`
- `source`

### 4.2 Gravity Pool

`Gravity Pool` 是一组筛选条件的持久化快照，不是文件夹，也不是静态集合。

它保存的是“观察视角”，不是“知识所有权”：

- 用户可以一键回到某组关注主题。
- 系统保存的是过滤器，而不是 block 的归属关系。
- 当前 Pool 语境会贯穿 Feed、Write、Graph 与局部同化事件。

## 5. 当前三大场景

### 5.1 Feed

`/feed` 是知识流入口，当前负责：

- `Omni Filter` 全文搜索与标签交集筛选
- `Quick Capture` 吸顶式输入流
- Grid / List 双视图切换
- 基于引用与维度重叠推导关系
- 超长文本的前端切块入库与线程串联

### 5.2 Write

`/write` 是沉浸写作场，当前负责：

- 自动创建 block
- 自动保存标题与正文
- 管理主维度标签，并以低存在感方式补充 `stage / source`
- 挂载 TipTap 与 `Adaptive Lens`
- 提供右侧 `Peek Drawer`
- 提供段落级 `Phantom Weaving` 与 `Crystal Assimilation` 的最小闭环

### 5.3 Graph

`/graph` 是关系总览，当前负责：

- 以网络视角投影扁平知识块
- 展示引用关系与维度重叠
- 帮助判断知识密度、线程聚集与结构张力

## 6. 当前已落地能力

截至当前代码基线，以下能力已经进入实现态：

- Zustand + persist 本地持久化
- Feed 的 `Omni Filter`、`Quick Capture`、Grid/List 视图切换
- Quick Capture 的 AI 自动命名与主维度打标
- `Semantic Auto-Chunking & Threading`
- Editor 自动保存、标签管理、AI 重新审视
- `@` 提及搜索与 `Adaptive Lens` 只读引用节点
- 右侧 `Peek Drawer` 并屏参考阅读
- 段落级 `Phantom Weaving`：`2500ms debounce + 当前段落 + Entity Lexicon + Fuse.js`
- `Crystal Assimilation` 的预览确认流、版本历史、恢复与来源追溯
- Graph 视图的 `Semantic Zoom + Spotlight Search + Cluster Framing`
- `activePoolContext`、`recentPoolEvents` 与 `recentAiTasks` 形成第一轮跨视图连续体验
- 首次冷启动且本地尚无持久化 `fluxBlocks` 时自动注入 onboarding seed 数据

## 7. Roadmap 总原则

从本文件开始，Conflux 不再保留游离的“未完成事项”列表。

所有待解决问题都必须被明确发配到以下几段版本里程碑之一：

- `v1.1`：智能攻坚期，只解决“主动感知与碎片吸收是否足够聪明”的问题。
- `v1.2`：创作体验期，只解决“编辑器是否足够成熟、可承载长文与多媒体”的问题。
- `v1.3`：信息流管网期，只解决“用户是否能驾驭规模化知识流”的问题。
- `v2.0 - v2.3`：桌面原生落地期，只解决“数据与心流是否真正进入操作系统层”的问题。
- `v3.0 - v3.1`：本地算力下沉期，只解决“重型任务与 AI 能力是否彻底脱离浏览器主线程”的问题。

没有版本归属的事项，视为不允许进入开发排期。

## 8. 演进路线图

### 8.1 `v1.1` 智能攻坚期

副标题：`Semantic Retrieval`

目标：

- 让系统的主动感知能力突破当前 `Fuse.js + 词典` 的查准率上限。

核心交付：

1. `混合召回引擎 (Hybrid Search)`
   - 在本地或极轻量级前端环境中引入 Embedding 检索。
   - 推荐链路从单纯模糊匹配升级为“Entity Lexicon + Embedding 相似度”的混合召回。
   - 新引擎必须能理解同义词、近义词与轻度语义变体。

版本结案标准：

- `Phantom Weaving` 对近义表达具备更稳定命中能力。
- 推荐结果仍遵守“先本地预筛，再有限调用模型”的漏斗纪律。

### 8.2 `v1.2` 创作体验期

副标题：`Rich Media & Block Ergonomics`

目标：

- 让编辑器在长文与结构化创作场景下具备成熟门槛。
- 收束当前 Web 阶段仍合理的创作体验增强项，而不再继续为浏览器存储瓶颈做局部补丁。

核心交付：

1. `动态大纲与折叠 (Outliner & Fold)`
   - 在主编辑器与 `Peek Drawer` 中补齐 Heading 大纲导航与局部跳转。
   - 当前进度：`/write` 已完成左侧 Outliner、平滑滚动跳转、`Scroll Spy`、超长标题截断、无标题空状态与跳转反馈；`Peek Drawer` 也已具备文内 TOC 与同层级导航语义。
   - 标题折叠已进入 `TipTap / ProseMirror Decoration` 轻量原型态：当前仅通过装饰器隐藏评估交互价值，不改动保存用内容结构，也不引入重型自定义节点。
2. `媒体边界校正`
   - 纯 Web 媒体承载路线已明确废弃，不再继续围绕 `IndexedDB` 做体验补丁。
   - 媒体能力统一移交 `v2.0 Native Persistence & Media`。

当前未完成开发清单：

`v1.2` 当前已按阶段结案，不再保留未完成清单；后续若继续打磨长文体验，应视为 `v1.3` 或更后续阶段的增强，而不是 `v1.2` 尾项。

版本结案标准：

- `大纲体验收口`
  - 支持当前阅读位置的标题高亮追踪（Scroll Spy）。
  - 点击跳转后，正文有清晰的瞬时视觉反馈。
  - 超长标题会被优雅截断，无标题文档会显示低存在感空状态提示。
- `长文信息降噪`
  - 主编辑器与 `Peek Drawer` 的大纲 UI 层级已经统一收口。
  - 大纲体验仍符合 `Zen Canvas` 的低压迫基线。
- `折叠能力极简原型`
  - 已完成一种基于 `DecorationSet` 的轻量标题块折叠原型。
  - 没有改动 TipTap 的底层数据模型，仅作为未来是否值得正式引入的评估依据。
- `边界纪律`
  - 不再继续推进基于纯 Web `IndexedDB` 的媒体存储方案。
  - 编辑器复杂度提升后仍不破坏当前自动保存、引用链路与 Drawer 阅读体验。
- `当前结案说明`
  - 本轮按代码主链、文档同步与 `npm run lint` / `npm run build` 的通过结果宣布 `v1.2` 结案。
  - 人工体感验收延后到下一阶段回归，不再作为当前里程碑阻塞项。

### 8.3 `v1.3` 信息流管网期

副标题：`Advanced Filtering & Canvas`

目标：

- 让用户开始驾驭规模化知识堆积，而不只是继续堆笔记。
- 为 Feed / List / Graph 之外预留更强的人为编排与批量整理能力。
- 但在桌面原生底座稳定前，这一阶段整体降级为视图扩展储备，而非最近一轮优先战役。

核心交付：

1. `白板视图 (Infinite Canvas / Whiteboard)`
   - 引入 `tldraw` 或同类框架提供自由拖拽二维看板。
   - 用户可将知识块手动钉在看板上，建立临时或长期的空间化布局。
2. `高级属性看板 (Advanced Properties)`
   - 放开维度的高级手动管理入口。
   - 提供表格视图，用于批量整理知识块属性与元数据。
   - 让 `stage / source / domain / format / project` 进入可批量操作的治理层。

版本结案标准：

- 用户可以通过 Canvas 进行空间化整理，而不是只依赖 Feed 与 Graph。
- 用户可以通过表格视图进行批量属性治理。
- 高级整理能力不会破坏现有轻量输入体验。

### 8.4 `v2.0` 存储纪元

副标题：`Native Persistence & Media`

目标：

- 彻底放弃以纯 Web `IndexedDB` 解决容量瓶颈的设想。
- 让 Conflux 的状态与富媒体第一次真正物理降落到桌面文件系统与原生持久化层。

核心交付：

1. `Native Persistence`
   - 使用 Tauri Plugin Store（JSON）或 SQLite 替换浏览器 `localStorage`。
   - 为 `fluxBlocks`、`savedPools` 与关键运行时状态建立异步落盘能力。
2. `Native Media Pipeline`
   - 接管系统剪贴板、拖拽与文件系统 API。
   - 用户粘贴图片时，直接将资源保存到本地隔离目录，例如 `~/.conflux/media/`。
   - TipTap 正文只渲染本地资源标识符或路径引用。

当前进度：

- `Native Persistence`
  - `Tauri Store` 已接入主链，Zustand persist 已切换为异步桌面存储适配。
  - 首启支持从 legacy `localStorage` 自动迁移并落盘到 `conflux_universe.json`。
  - 当前桌面端已完成真实落盘验证，说明原生持久化路径已经进入实现态。
- `Native Media Pipeline`
  - `plugin-fs` 与本地 `media/` 目录 scope 已接通。
  - `FluxEditor` 已能在 Tauri 环境下拦截图片粘贴/拖拽并写入本地目录，再回填安全 URL。
  - 这条链目前仍需继续收口，尚未单独构成 `v2.0` 结案。

当前未完成开发清单：

1. `桌面持久化收口`
   - 验证 `Tauri Store` 作为桌面主存储路径的稳定性与恢复边界。
   - 明确迁移失败、空文件、损坏文件时的处理策略。
2. `原生媒体闭环`
   - 让图片/附件链路从“能写入本地”推进到“可重启恢复、可稳定引用”的完整主路径。
   - 明确媒体缺失、删除或路径失效时的退化表现。
3. `桌面与 Web 边界`
   - 继续澄清 Tauri 桌面主路径与 Web fallback 的职责边界。
   - 避免将壳层接入误判为桌面版本已完成。
4. `验收与结案口径`
   - 补齐桌面实机下的存储、媒体、恢复链路验收。
   - 保证文档、代码与实际桌面行为一致。

建议验收清单：

- `持久化迁移与恢复`
  - 验证 legacy `localStorage` 到 `conflux_universe.json` 的迁移。
  - 验证最近编辑在窗口关闭/隐藏后的恢复结果。
- `异常载荷恢复`
  - 验证损坏 `localStorage / Tauri Store` 载荷时不会阻塞冷启动。
  - 验证坏数据会被备份并回退到安全默认值。
- `图片/附件闭环`
  - 验证图片与普通附件都能本地落盘、重启恢复、缺失退化。
  - 验证编辑期删除、删整篇笔记与启动时扫描三类孤儿媒体回收都正常工作。
- `Web fallback`
  - 验证浏览器环境仍使用 `localStorage + Data URL` 降级路径，且不因桌面逻辑崩溃。

版本结案标准：

- 浏览器存储配额不再是 Conflux 的主约束。
- 图文混排依托桌面原生文件系统，而不是 `IndexedDB` 临时补丁。
- 富媒体、状态与正文引用关系形成统一的桌面持久化模型。
- 桌面端持久化与媒体链路已成为稳定主路径，而不只是实验接线。
- Tauri 桌面路径与 Web fallback 的行为边界清楚，且文档保持一致。
- 上述桌面实机验收清单至少完成一轮，并能复述验证结论。

### 8.5 `v2.1` 桌面心流期

副标题：`OS-Level Ergonomics`

目标：

- 让 Conflux 从“桌面窗口”演进为“操作系统常驻工具”。
- 让 Quick Capture 进入真正低摩擦、低抢焦点的桌面心流。

核心交付：

1. `Global Quick Capture`
   - 注册系统级全局快捷键，例如 `Cmd/Ctrl + Shift + Space`。
   - 在任意第三方应用之上呼出极简速记胶囊。
2. `Tray & Silent Guard`
   - 引入系统托盘入口。
   - 在后台维持轻量任务、词典预热与后续异步队列宿主。

版本结案标准：

- 用户可以在主窗口关闭或隐藏时继续快速捕获内容。
- Conflux 具备桌面级驻留与唤起语义。
- 桌面工作流不再依赖浏览器标签页驻场。

### 8.6 `v2.2` 视图扩张期

副标题：`Advanced Views`

目标：

- 在桌面原生底座稳定后，重启原 `v1.3` 中被延后的高密度视图扩展。
- 让 Canvas 与数据库式治理建立在更可靠的本地存储和性能边界之上。

核心交付：

1. `Infinite Canvas`
   - 引入 `tldraw` 或同类高性能二维引擎。
   - 支持将知识块自由钉在桌面级无限白板中。
2. `Table / Properties View`
   - 提供数据库式视图。
   - 支持对 `domain / format / project / stage / source` 做批量治理、筛选与整理。

版本结案标准：

- 原 `v1.3` 的视图扩展正式转入桌面原生时代落地。
- 视图层扩张不再受浏览器存储与单线程压力掣肘。
- Canvas 与属性治理成为规模化知识流的第一层桌面工作台。

### 8.7 `v2.3` 算力下沉期

副标题：`Rust Backend & Local LLM`

- 让前端退回为纯渲染层，把真正重的任务交给桌面后端。
- 让 AI 能力同时拥有云端 BYOK 与本地离线运行两条路径。

核心交付：

1. `Rust Multi-thread Runtime`
   - 将长文 `Semantic Auto-Chunking`、全库嗅探与重型检索任务下沉到 Tauri Rust 后端。
   - 为前端提供更稳定的 IPC 调度接口。
2. `Local LLM Integration`
   - 在现有 BYOK 之外，原生接入 Ollama 等本地模型运行时。
   - 支持离线 AI 打标、同化与生成链路。

版本结案标准：

- 重型任务不再由前端主线程独自承担。
- 桌面后端成为检索、切块与 AI 能力的第一宿主。
- Conflux 开始形成“桌面原生存储 + 桌面原生算力 + 本地模型可选”的完整闭环。

版本结案标准：

- 关键 AI 能力在离线环境中具备可运行路径。
- 用户可以在本地模型与云端模型之间按场景切换。
- Conflux 形成“本地数据、本地算力、本地存储”的完整堡垒形态。

## 9. 待决问题到版本归属

当前所有仍未收口的问题，统一归属如下：

- `Phantom Weaving` 的语义化升级：归 `v1.1`
- 大纲导航与长文编辑人体工学收口：归当前 `v1.2` 基线
- 原 `v1.2` 的媒体承载需求已并入 `v2.0`
- 原 `v1.3` 的 Canvas 与高级属性治理已延后并并入 `v2.2`
- `stage / source` 的完整治理层：从 `v1.3` 开始进入成熟形态
- 桌面持久化迁移与原生多媒体：归 `v2.0`
- 全局快捷键、托盘与桌面心流：归 `v2.1`
- Canvas 与 Database 视图扩张：归 `v2.2`
- Rust 后端下沉与本地大模型：归 `v2.3`

自此不再保留游离的“以后再看”条目。
