# Conflux MVP 主指令

副标题：零摩擦输入、克制推荐与双向知识同化

## 1. 本文件的作用

这份文档不是版本日志，而是当前阶段所有开发都必须服从的产品纪律。

它要解决的问题只有一个：无论代码和文档经历过怎样的迭代漂移，Conflux 都不能被做回“目录驱动的笔记软件”或“边写边骚扰的智能编辑器”。

## 2. 版本校正说明

当前代码基线已经同时包含：

- `v0.9` 的段落级本地推荐漏斗
- `v1.0` 的长文语义切块与线程串联

因此，本文件面向的是 `v1.0 稳定化阶段`，以及已经被版本化约束的后续路线图。

## 3. 北极星原则

### 3.1 主动预期

系统要在后台静默理解用户当前正在写什么，并主动发现可能相关的知识块，而不是永远等待用户显式输入命令。

### 3.2 零压迫 UI

界面必须继续遵守 `Zen Canvas`：浅灰背景、白色卡片、大留白、低噪声交互。智能提示应该像呼吸，而不是像警报。

### 3.3 双向生长

知识引用不应只有单向“摘取”。真正的个人外脑，需要支持“当前新洞察反向同化回旧知识母体”。

### 3.4 本地优先漏斗

默认流程必须是：

1. 本地轻量预筛
2. 隐蔽 UI 浮现
3. 用户主动确认
4. 只在必要时调用模型

这条纪律同时适用于写作推荐与长文入库。

## 4. 当前已确定的主链

截至当前代码基线，Conflux 的主链已经收束为：

`Gravity Pool -> Feed -> Quick Capture 长文切块 -> Write 段落嗅探 -> Peek Drawer 核对 -> Crystal Assimilation -> Graph`

这里真正已经成立的事实是：

- `Phantom Weaving` 不再做字级虚线或 Hover Card，而是只对当前段落做本地高置信嗅探，并在右下角浮出单一提示。
- `Crystal Assimilation` 只有在用户从 Drawer 主动确认后，才会调用模型并进入差异预览。
- `Adaptive Lens` 已稳定收束为显式引用节点，不再承担边写边摘要职责。
- `Quick Capture` 已具备长文 `Semantic Auto-Chunking & Threading` 能力。
- `/graph` 已完成模块化拆分，并具备 `Semantic Zoom + Spotlight Search + Cluster Framing`。

## 5. 当前最重要的纪律

### 5.1 不要求用户先知道要 `@` 谁

推荐链路可以主动预期，但主动预期必须是低压迫的。

### 5.2 不为了“智能感”频繁打断正文

系统可以理解上下文，但不应通过正文内侵入式交互破坏心流。

### 5.3 不让巨量输入污染知识库

长文必须先被本地切成可渲染、可检索、可追踪的小块，再进入主数据层。

### 5.4 宁可漏报，也不要误报

无论是实体嗅探还是长文切块，优先级永远是稳定、可解释、低噪音。

## 6. 版本化执行纪律

从现在开始，所有待决事项都必须纳入明确版本：

- `v1.1` 只负责主动感知升级与 Hybrid Search。
- `v1.2` 只负责编辑器结构化创作体验与长文人体工学。
- `v1.3` 只负责 Canvas、表格视图与高级属性治理。
- `v2.0` 只负责桌面原生持久化与多媒体落盘。
- `v2.1` 只负责系统级入口与桌面心流。
- `v2.2` 只负责桌面高级视图扩张。
- `v2.3` 只负责后端算力下沉与本地模型接入。

如果一个任务无法归属到这些版本之一，就不应开始开发。

## 7. 对未来 Agent 的执行要求

如果后续把本项目交给其他代码 Agent，请优先遵守以下顺序：

1. 先读 [`06-HANDOFF.md`](./06-HANDOFF.md)、[`04-CHANGELOG.md`](./04-CHANGELOG.md)、[`02-ARCHITECTURE.md`](./02-ARCHITECTURE.md)
2. 先确认当前代码属于 `v1.0 稳定化阶段`
3. 任何待解决问题都必须先归属到 `v1.1`、`v1.2`、`v1.3`、`v2.0`、`v2.1`、`v2.2` 或 `v2.3`
4. 不要把 `v0.8`、`v0.9`、`v1.0` 误读为三次干净切开的独立发布
5. 所有新的智能链路都应继续遵守“本地预筛 -> 用户确认 -> 精确调用模型”

## 8. 当前阶段结论

Conflux 现在最大的风险已经不是“没有能力”，而是：

- 文档与代码再次漂移
- 路线图失去约束力
- 零散 TODO 重新回到开发语境

所以当前阶段最应该守住的，不是更快往后跳版本，而是让所有演进都先进入被约束的版本沙盘。
