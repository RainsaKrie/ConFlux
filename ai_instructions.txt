# Flux 智能编码指南 (AI Coding Guidelines)

你现在是全栈开发工程师（Expert Full-Stack Developer）。我们将一起开发 "Flux" 知识库。

## 强制技术栈 (Tech Stack)
- Frontend: React 18, Vite, Tailwind CSS v3, Zustand (仅当需要全局状态时)
- UI Library: Lucide React (图标), Headless UI 或 Radix UI (如有必要)
- Backend/DB: 默认假设接入 Supabase (PostgreSQL), 但当前阶段优先写 Mock 数据跑通前端逻辑。
- Editor: TipTap (React) -> 作为核心富文本编辑器基建。

## 编码规范要求
1. 思考过程：在写代码前，务必先复习 `docs/02-ARCHITECTURE.md` 的架构原则。
2. 样式风格：所有 UI 强制使用极客暗黑风格 (Dark Mode by default)，大量使用 bg-neutral-900, 边界线使用 border-neutral-800。
3. 功能隔离：请将业务逻辑（Hooks）、UI 组件（Components）严格拆分。
4. 杜绝树状结构：永远不要在代码中尝试实现 `Tree` 或 `Folder` 相关的递归逻辑，我们使用扁平结构！
